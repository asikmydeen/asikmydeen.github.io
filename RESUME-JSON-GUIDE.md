# Resume JSON Data Structure

This document explains the JSON-based resume data loading system. The resume data is now completely decoupled from the HTML structure, allowing for easy updates and the ability to support any person's resume.

## Overview

The resume website now loads all data dynamically from `resume.json` at runtime. This provides:

1. **Data/View Separation**: Resume data is completely separated from the presentation layer
2. **Easy Updates**: Update your resume by simply editing the JSON file
3. **Reusability**: Can be adapted for anyone's resume by changing the JSON data
4. **Maintainability**: Clean architecture with modular components

## File Structure

```
asikmydeen.github.io/
├── resume.json          # Your resume data (edit this!)
├── resume-loader.js     # Data loading and rendering module
├── app.js              # Interactions and export functions
├── styles.css          # Styling
├── index.html          # Minimal HTML shell (generated by build.sh)
└── build.sh            # Build script
```

## JSON Schema

### Top-Level Structure

```json
{
  "personal": { ... },
  "summary": { ... },
  "skills": [ ... ],
  "projects": [ ... ],
  "experience": [ ... ],
  "education": [ ... ]
}
```

### Personal Information

```json
{
  "personal": {
    "name": "Your Full Name",
    "title": "Your Job Title",
    "phone": "+1234567890",
    "email": "your.email@example.com",
    "location": {
      "city": "City",
      "state": "ST",
      "country": "US"
    },
    "links": [
      {
        "type": "linkedin|github|project|website",
        "icon": "emoji",
        "label": "Display Text",
        "url": "https://..."
      }
    ]
  }
}
```

### Professional Summary

```json
{
  "summary": {
    "text": "Your professional summary paragraph...",
    "highlights": [
      {
        "icon": "emoji",
        "text": "Key Strength"
      }
    ],
    "metrics": [
      {
        "value": 12345,
        "label": "Description",
        "suffix": "+" // Optional: "+", "%", etc.
      }
    ]
  }
}
```

### Skills

```json
{
  "skills": [
    {
      "category": "Skill Category Name",
      "icon": "emoji",
      "proficiency": 95, // 0-100
      "items": [
        "Skill 1",
        "Skill 2"
      ]
    }
  ]
}
```

### Projects

```json
{
  "projects": [
    {
      "title": "Project Name",
      "icon": "emoji",
      "link": { // Optional
        "text": "Link Text",
        "url": "https://..."
      },
      "description": "Project description (HTML allowed: <strong>, etc.)",
      "tags": [
        "Tag 1",
        "Tag 2"
      ],
      "details": [
        "Technical detail 1",
        "Technical detail 2"
      ]
    }
  ]
}
```

### Experience

```json
{
  "experience": [
    {
      "title": "Job Title",
      "company": "Company Name",
      "location": "City, State",
      "period": "Mon YYYY – Mon YYYY",
      "ariaLabel": "Full date range for accessibility",
      "highlights": [ // Optional
        {
          "icon": "emoji",
          "text": "Key Achievement"
        }
      ],
      "responsibilities": [
        "Responsibility or achievement 1",
        "Responsibility or achievement 2"
      ]
    }
  ]
}
```

### Education

```json
{
  "education": [
    {
      "degree": "Degree Name",
      "institution": "Institution Name",
      "cgpa": "GPA/CGPA",
      "graduationDate": "Month Year"
    }
  ]
}
```

## How to Update Your Resume

1. **Edit `resume.json`**: Update your data in the JSON file
2. **Run build script**: Execute `bash build.sh`
3. **Test locally**: Open `index.html` in a browser
4. **Deploy**: Push changes to GitHub (if using GitHub Pages)

## Features

### Animations
- Fade-in animations for sections
- Counter animations for metrics
- Skill bar animations
- Project card expand/collapse

### Export Functions
- **PDF Export**: Opens print dialog for saving as PDF
- **Word Export**: Downloads resume as .docx file
- **Print**: Direct printing support

All export functions work with the dynamically loaded JSON data.

### Accessibility
- ARIA labels and landmarks
- Semantic HTML structure
- Keyboard navigation support
- Schema.org structured data

## Customization

### Using Different JSON Files

To use a different JSON file or support multiple resumes:

```javascript
// In your HTML, before loading resume-loader.js:
<script>
  window.RESUME_JSON_PATH = 'path/to/your/resume.json';
</script>
```

Or modify `resume-loader.js`:

```javascript
const loader = new ResumeLoader('custom-resume.json');
```

### Styling

All styles are in `styles.css`. The JSON-based system uses the same CSS classes as before, so existing styles continue to work.

## Browser Compatibility

- Modern browsers (Chrome, Firefox, Safari, Edge)
- Requires JavaScript enabled
- Uses Fetch API for loading JSON

## Technical Details

### Loading Process

1. `index.html` loads with minimal HTML structure
2. `resume-loader.js` executes on DOM ready
3. `ResumeLoader` class fetches `resume.json`
4. Data is rendered into HTML using template strings
5. Event listeners and animations are initialized
6. Export functions remain available

### Data Validation

The system expects all required fields to be present in the JSON. Missing optional fields (like `highlights` or `link`) will be gracefully handled.

## Migration from Static HTML

If you had a static HTML resume before:

1. Your data has been extracted to `resume.json`
2. The `partials/` directory is no longer used
3. `build.sh` now generates a minimal HTML shell
4. All export and animation features continue to work

## Troubleshooting

### Data Not Loading

- Check browser console for errors
- Verify `resume.json` is valid JSON (use a JSON validator)
- Ensure the JSON file is in the same directory as `index.html`
- Check for CORS issues if loading from file://

### Animations Not Working

- Ensure `app.js` is loaded after `resume-loader.js`
- Check that animation initialization functions are called
- Verify CSS is loaded properly

### Export Functions Not Working

- Ensure `downloadPDF()` and `downloadWord()` functions are defined in `app.js`
- Check that export buttons have correct `onclick` attributes
- Verify `.no-print` elements are properly excluded
